Промпт для первоначальной версии скрипта без изменений.

Ты эксперт по Linux, Bash и Docker с практическим опытом DevOps.

Нужно написать production-ready bash-скрипт для установки и настройки HTTP и SOCKS5 прокси (3proxy) в Docker.

Требования:

1. Общие требования

* Скрипт должен быть полностью автономным (one-shot install)
* Должен работать на Debian/Ubuntu
* Использовать `set -euo pipefail`
* Код должен быть устойчив к ошибкам (учесть поведение set -e)
* Добавить понятные цветные сообщения (msg/ok/warn/err)

2. Docker

* Проверить установлен ли Docker, если нет — установить
* Проверить наличие `docker compose`
* Использовать образ: 3proxy/3proxy:latest
* Контейнер должен называться: 3proxy
* Использовать docker-compose.yml

3. Идемпотентность (очень важно)

* Если контейнер уже существует:

  * если запущен — остановить
  * удалить контейнер
* Затем пересоздать заново
* Проверку делать до запуска и до проверки портов

4. Порты

* Порты задаются переменными в начале:
  HTTP_PORT
  SOCKS_PORT
* Проверить занятость портов через ss
* Если порт занят — завершить скрипт с ошибкой

5. Конфигурация 3proxy

* Создать /opt/3proxy/conf/3proxy.cfg
* Логи: /opt/3proxy/logs
* Настроить:

  * proxy (HTTP)
  * socks (SOCKS5)
* auth strong
* users "user:CR:HASH"

6. Пароль

* В начале скрипта:
  PROXY_USER
  PROXY_PASS
* Если PROXY_PASS пустой:

  * сгенерировать безопасный пароль
  * использовать только URL-safe символы (A-Za-z0-9._-)
* Хранить в конфиге только хэш:
  openssl passwd -1
* В конце скрипта:

  * вывести пароль пользователю
* В начале скрипта:

  * вывести предупреждение удалить пароль из скрипта

7. UFW

* Открыть HTTP_PORT и SOCKS_PORT
* Перезагрузить ufw

8. Healthcheck

* Сделать без nc/curl
* Использовать /proc/net/tcp и tcp6
* Проверять порт в hex формате

9. Проверка запуска

* Проверить, что контейнер реально запущен
* Если нет — вывести docker logs и завершить

10. Вывод результата
    В конце вывести:

* IP сервера
* порты
* логин/пароль
* пути к конфигам

11. Тестирование
    Вывести готовые команды:

* curl HTTP proxy
* curl SOCKS proxy (socks5h)

12. Безопасность и UX

* Не использовать интерактивный ввод
* Пароль не должен ломать URL
* Учитывать pipefail (не использовать конструкции, которые дают SIGPIPE)
* Скрипт должен быть читаемый и разбит на функции

13. Дополнительно

* Добавить комментарии в коде с важными замечаниями:

  * про пароль
  * про URL encoding
  * про UFW
  * про логи

Выведи полностью готовый финальный скрипт без пояснений.
